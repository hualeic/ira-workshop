## 3. Sidebar 会话管理

> 对齐 `05` US-001。描述会话列表的增删查交互。

| 功能 | 触发 | API 调用 | 行为 |
|------|------|----------|------|
| 加载列表 | 页面渲染 | `GET /sessions` | 渲染会话列表 + 默认选中第一个 |
| 新建会话 | 点击"+ 新建" | `POST /sessions` | 插入列表头部 + 自动选中 |
| 删除会话 | 点击 × | `DELETE /sessions/<id>` | 确认弹窗 → 从列表移除 |
| 选中会话 | 点击会话条目 | `GET /sessions/<id>/records` | 高亮当前会话 + 加载该会话的问答记录到 Main 区域 |

## 4. Main 内容区（三态渲染）

> 对齐 `05` US-002（问答提交）、US-003（历史记录）。描述三种条件下显示什么。

| 状态 | 条件 | 显示内容 |
|------|------|----------|
| **A 空状态** | `currentSession === null` | "请创建或选择一个会话开始" |
| **B 常见问题** | 有会话 && 无记录 | 显示 4~6 个常见投研问题卡片网格（如"今日市场概况""热门研报摘要"），点击后自动填入输入框并发送提问 |
| **C 对话历史** | 有记录 | 按时间正序渲染记录卡片，每条包含：用户提问文本、AI 回答文本、来源标签（CoPaw/百炼/离线演示）、响应耗时(ms)、时间戳 |

### 4.1 记录卡片中的来源标签逻辑

> 根据 `answer_source` 和 `llm_used` 组合显示不同标签。

| `answer_source` | `llm_used` | 显示文本 | 样式 |

| `copaw` | `true` | CoPaw | 绿色/蓝色标签 |
| `bailian` | `true` | 百炼 | 绿色/蓝色标签 |

## 6. 错误处理

| 后端 `error.code` | HTTP | 前端展示 |
|--------------------|------|----------|
| `EMPTY_QUERY` | 400 | 错误提示："请输入问题" |
| `INVALID_QUERY` | 400 | 错误提示："问题过长" |
| `UPSTREAM_ERROR` | 500 | 错误提示："服务暂时不可用，请稍后重试" |
| `SESSION_NOT_FOUND` | 404 | 错误提示："会话不存在，请刷新页面" |
| 网络异常 | — | 错误提示："网络连接失败，请检查网络后重试" |

## 7. 状态管理

> React `useState` Hooks，列出你的组件需要管理哪些状态。

| State 变量 | 类型 | 初始值 | 用途 |
|------------|------|--------|------|
| `sessions` | `Session[]` | `[]` | 会话列表 |
| `currentSession` | `Session\|null` | `null` | 当前选中会话 |
| `records` | `QARecord[]` | `[]` | 当前会话的问答记录列表 |
| `query` | `string` | `""` | 输入框中的用户提问文本 |
| `loading` | `boolean` | `false` | 是否正在等待 API 响应（控制发送按钮禁用） |
| `error` | `string\|null` | `null` | 当前错误提示信息 |
| `capabilities` | `object\|null` | `null` | 能力探测结果（CoPaw/百炼配置状态） |
| `confirmDelete` | `string\|null` | `null` | 待确认删除的 session_id |
