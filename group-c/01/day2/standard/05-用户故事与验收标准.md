> 每条故事 **同一文档内** 写全：**US + AC + REQ**。

## US-M1-001（P0）消息流与分页

| 字段 | 内容 |
|------|------|
| **编号** | US-M1-001 |
| **故事** | 作为投研用户，我希望按时间浏览投研消息并翻页，以便早会前掌握最新动态。 |
| **AC** | 1) `GET /research/messages` 返回 `items`、`nextCursor`、`hasMore`，顺序为发布时间 **新→旧**；2) `limit` 非法返回 **400** `M1_VALIDATION_ERROR`；3) `unreadOnly=true` 时仅含 `read=false` 项；4) 响应与 `09` §2.1 字段一致。 |
| **REQ** | REQ-M1-001 → `14` 矩阵 |

## US-M1-002（P0）详情与已读

| 字段 | 内容 |
|------|------|
| **编号** | US-M1-002 |
| **故事** | 作为投研用户，我希望打开消息详情并标记已读，以便区分已跟进与待处理。 |
| **AC** | 1) `GET /research/messages/{id}` 返回 `09` §2.2 `MessageDetail`；2) `PATCH .../read` 幂等，重复调用 **200** 且 `read=true`；3) 无效 id **404** `M1_MESSAGE_NOT_FOUND`；4) 已读后列表同条 `read` 为 **true**（刷新或乐观更新均可测）。 |
| **REQ** | REQ-M1-002 |

## US-M1-003（P0）搜索

| 字段 | 内容 |
|------|------|
| **编号** | US-M1-003 |
| **故事** | 作为投研用户，我希望按关键词搜索历史消息，以便快速定位标的或事件相关条目。 |
| **AC** | 1) `q` 为空或过短返回 **400**；2) 返回结构与列表相同包络，且命中项可含 `highlight`；3) `from`/`to` 非法日期 **400**；4) 超限触发 **429** 时 body 含 `M1_RATE_LIMIT`。 |
| **REQ** | REQ-M1-003 |
