## 1. 逻辑架构（M1 单体边界）

```
React SPA ──HTTP──► Flask  research_bp  (/api/v1/research/*)
                         │
                         ├──► PostgreSQL（research_messages / research_message_reads）
                         └──► （预留）未来 MessageIngest 适配器 ◄── M2 作业，M1 首版可不部署
```

- **M1 首版**：无独立消息网关；Flask 直接读库。  
- **预留**：引入 `MessageIngest` 服务或队列时，**仅替换写入侧**；`09` 读 API **保持不变**。

## 2. 技术选型

| 层 | 选型 | 说明 |
|----|------|------|
| 前端 | React + React Router | 与全项目一致 |
| 后端 | Flask | Blueprint `research_bp` |
| 持久化 | PostgreSQL | `10` §2 |
| 缓存 | 无（P0） | P1 可选 Redis |

## 3. 部署与环境

- `dev` / `staging` / `prod`；数据库连接串与 `DB_POOL` 配置经环境变量注入。  
- **不**在 M1 仓库内硬编码兄弟模块 URL；后续集成用配置中心或 `INTEGRATION_*` 环境变量。
