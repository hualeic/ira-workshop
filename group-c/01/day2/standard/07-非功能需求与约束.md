## 1. 性能与可用性

### 1.1 API 延迟目标

| 端点 | P95 目标 | 说明 |
|------|----------|------|
| `GET /research/health` | **≤ 200 ms** | |
| `GET /research/messages` | **≤ 500 ms** | `limit≤50` |
| `GET /research/messages` | **≤ 1200 ms** | `limit=100` |
| `GET /research/messages/search` | **≤ 1500 ms** | `limit=20`，数据量 **≤ 5 万条** 消息量级 |
| `GET /research/messages/{id}` | **≤ 400 ms** | |
| `PATCH .../read` | **≤ 300 ms** | 单次 upsert |

*早会高峰**：列表 QPS 目标在 `12` 压测中验证；若超目标，加只读副本或缓存热点页（**不**在 M1 首版强制）。*

### 1.2 可用性

| 环境 | 目标 |
|------|------|
| prod | 月度 **≥ 99.5%**（本模块 API 可用，不含全局网关） |

### 1.3 降级

- DB 超时：列表返回 `503` + 可重试提示；**禁止** 返回半页脏数据。  
- 搜索过载：优先 **429**（见 `09`），或降级为仅标题搜索（须配置开关并记录日志）。

---

## 2. 安全与合规

- 展示内容 **不构成投资建议**；免责声明与 `04` 一致。  
- **鉴权**：所有端点需登录；`messageId` 越权返回 **404**（防枚举）。  
- **日志**：禁止记录 `body` 全文与搜索词明文（可 hash 或截断）；须带 `traceId`、`module=M1`。

---

## 3. 可观测性

### 3.1 日志字段（必选）

`traceId`、`requestId`、`userId`（脱敏）、`module=M1`、`path`、`durationMs`、`statusCode`；可选 `messageId`（仅详情/已读）。

### 3.2 指标建议

| 指标 | 类型 | 说明 |
|------|------|------|
| `m1_http_requests_total` | Counter | `path`, `status` |
| `m1_http_request_duration_seconds` | Histogram | `path` |
| `m1_search_total` | Counter | `result=ok\|error` |
| `m1_read_patch_total` | Counter | |

### 3.3 告警（prod）

| ID | 条件 | 级别 |
|----|------|------|
| M1-A1 | 本模块 5xx **> 2%** 持续 **5 min** | P2 |
| M1-A2 | `GET /messages` P95 **> 1.5 s** 持续 **15 min** | P2 |
| M1-A3 | 搜索 429 **> 20%** 持续 **10 min** | P3 |

可与 **阿里云 SLS** 按 `module:M1` 订阅；SQL 模式同 M5 `07` §3.4。
