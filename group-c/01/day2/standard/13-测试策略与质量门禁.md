## 2. 测试层级与覆盖对象

| 层级 | 工具建议 | 覆盖范围 | 最低要求（P0） |
|------|-----------|----------|----------------|
| **单元** | pytest / Jest | 游标编解码、`limit` 裁剪、`unreadOnly` SQL 条件、已读 upsert 幂等 | 核心纯函数与 Repository **≥ 80%** 行覆盖（本模块业务包） |
| **集成** | pytest + 测试库 | `GET /research/messages`、`search`、`{id}`、`PATCH read` + 真实 PG（或 Testcontainers） | 每条主路径 **≥1** 条集成用例 |
| **契约** | schemathesis / Dredd（可选） | `09` 响应 JSON 与 HTTP 码 | PR 可选；发版前 **至少手跑** Postman 集合 |
| **E2E** | Playwright / Cypress | 浏览器：收件箱 → 详情 → 已读 → 搜索 | **`05` 中 US-M1-001～003** 各 ≥1 条 |

### 2.1 单测用例清单（示例 ID）

| TC-ID | 断言要点 |
|-------|-----------|
| M1-U-CURSOR-01 | 相同 `items` 下 `nextCursor` 解析后下一页首条 `seq` 递增且不漏不重 |
| M1-U-READ-01 | 同一 `messageId` 两次 `PATCH read` 不产生重复 `research_message_reads` 行 |
| M1-U-SEARCH-01 | `q` 空串在服务端拒绝（与 `09` 400 一致） |

### 2.2 集成用例清单（示例 ID）

| TC-ID | `09` 路径 | 断言要点 |
|-------|-----------|-----------|
| M1-I-LIST-01 | `GET /research/messages` | `unreadOnly=true` 时全部 `read=false` |
| M1-I-SEARCH-01 | `GET /research/messages/search` | `highlight` 可选；`from`/`to` 边界含入 |
| M1-I-DETAIL-01 | `GET /research/messages/{id}` | 404 与 403 口径与 `09` 一致 |
| M1-I-READ-01 | `PATCH .../read` | 200 幂等；随后 list 中 `read=true` |
