## 2. 前端路由与页面（React）

| 路由 | 页面职责 |
|------|-----------|
| `/research/inbox` | 消息流：`GET /research/messages`，游标翻页、`unreadOnly` / `category` 筛选 |
| `/research/search` | 搜索：`GET /research/messages/search`，展示 `highlight` |
| `/research/messages/:id` | 详情：`GET .../{id}`，进入页或抽屉内 **`PATCH .../read`** |

- 嵌入全局 Shell；页脚 **基金投研免责声明**（文案版本可配置，与 `04` 一致）。  
- 列表/详情字段 **严格** 与 `09` DTO 一致，便于 Mock Service Worker。

---

## 3. 状态与交互

| 状态 | 行为 |
|------|------|
| 列表 loading | 骨架屏；保留上一页数据时可显示 stale-while-revalidate |
| 空列表 | 区分「无数据」与「筛选无结果」文案 |
| 搜索空 | 提示缩短关键词；`q` 非法由接口 400 映射 Toast |
| 错误 | `traceId` 可复制；403/404 单独文案 |
| 已读 | `PATCH` 成功后乐观更新列表项 `read=true`；失败回滚 |

**P1**：SSE 推送新消息（本文不规定事件格式，避免绑定 M4）。

---

## 4. 后端行为（Flask）

- Blueprint：`research_bp`，`url_prefix=/api/v1/research`。  
- **游标**：编码/解码集中工具函数；排序键 **`published_at DESC, id DESC`** 与 `10` 索引一致。  
- **搜索**：实现路径见 `10` §3；须遵守 `09` 限流。  
- **已读**：`INSERT ... ON CONFLICT DO NOTHING` 或等价，保证幂等。  
- **权限**：P0 所有登录用户可读全部非删消息；行级权限为 P1，见 `10` 扩展。
