## 4. 变更流程（必须遵守）

1. 产品/架构提出变更 → 先改 **`05` AC**（若行为变）→ 再改 **`09`/`10`**。  
2. 同一 PR 内更新 **本表** 行与 **`13` TC-ID**（新增或调整断言）。  
3. 若改 `09` 对外 JSON **字段名/类型**，在 PR 描述中 **@ 依赖方**（M2/M4 README）。

## 5. 影响分析模板（复制到 PR）

```
- [ ] 是否改 DB 迁移：是/否（文件：…）
- [ ] 是否改公开 API：是/否（diff `09` §…）
- [ ] 影响模块：M2 / M4 / 无
- [ ] 回归：`13` §4 勾选项
```
