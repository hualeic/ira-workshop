## 1. 安全目标与合规要求

> 明确本模块需遵循的安全目标和外部合规标准。投研场景尤其关注数据隐私与内部信息防泄露。

| 安全目标 | 说明 | 对应合规 |
|----------|------|----------|
| 数据不出域 | 所有用户数据、研报内容、问答记录禁止传输至外部服务 | 内部信息安全规范 |
| 最小权限原则 | 每个组件只拥有完成任务所需的最低权限 | 安全设计最佳实践 |
| 可审计可追溯 | 所有关键操作（问答、会话管理、配置变更）均有 traceId 链路追踪日志 | `07` §3 可观测性需求 |

## 2. 认证与授权

> 定义用户身份验证方式和权限控制模型。写出认证流程和角色划分。

### 2.1 认证机制

| 项目 | 规格 |
|------|------|
| 认证方式 | 当前教学版：无认证（开放访问）；生产版规划：JWT Token + 内部 SSO 对接 |
| Token 有效期 | 教学版不适用；生产版：Access Token 30min，Refresh Token 7d |
| 密码策略 | 教学版不适用；生产版：最少 8 位，含大小写 + 数字，90 天强制更换 |
| 多因素认证 | 教学版不适用；生产版：管理员角色强制 MFA，分析师可选 |

### 2.2 角色与权限矩阵

| 角色 | 研报上传 | 研报阅读 | 问答交互 | 对比分析 | 系统管理 |
|------|----------|----------|----------|----------|----------|
| 分析师 | ✗ | ✓ | ✓ | ✓ | ✗ |
| 管理员 | ✓ | ✓ | ✓ | ✓ | ✓ |
| 访客（只读） | ✗ | ✓ | ✗ | ✗ | ✗ |

**API 端点级权限矩阵**（对齐 `09` §1 全部 6 个端点）：

| # | API 端点 | 方法 | 分析师 | 管理员 | 访客 | 来源 |
|---|----------|------|--------|--------|------|------|
| 1 | `/api/v1/agent/capabilities` | GET | ✓ | ✓ | ✓ | `09` §1 #1 |
| 2 | `/api/v1/agent/ask` | POST | ✓ | ✓ | ✗ | `09` §1 #2 |
| 3 | `/api/v1/agent/sessions` | GET | ✓ | ✓ | ✓ | `09` §1 #3 |
| 4 | `/api/v1/agent/sessions` | POST | ✓ | ✓ | ✗ | `09` §1 #4 |
| 5 | `/api/v1/agent/sessions/<id>` | DELETE | ✓（仅自己） | ✓（全部） | ✗ | `09` §1 #5 |

## 3. 数据安全

> 定义数据分类分级、加密策略、存储安全规范。**投研场景核心关注点**。

### 3.1 数据分类与分级

| 数据类型 | 安全等级 | 示例 | 保护措施 |
|----------|----------|------|----------|
| 研报原文 | **机密** | PDF/HTML 研报文件 | 加密存储，访问审计 |
| 问答记录 | **内部** | 用户提问与AI回答 | 存储加密，定期清理 |
| 用户信息 | **内部** | 用户名、角色、操作日志 | 脱敏展示，最小采集 |
| 系统配置 | 一般 | 模型参数、界面配置 | 版本控制 |

### 3.2 加密策略

| 场景 | 加密方式 | 说明 |
|------|----------|------|
| 传输加密 | TLS 1.2+ | 教学版 HTTP；生产版强制 HTTPS |
| 存储加密 | AES-256 | 教学版明文 JSON；生产版加密存储 |
| 密钥管理 | 环境变量（`.env`）；生产版升级至 KMS | 密钥不硬编码，生产版定期轮转 |

## 4. 接口安全

> 与 `09` API 规格配合，定义 API 层面的安全防护措施。

| 防护措施 | 实现方式 | 说明 | 来源 |
|----------|----------|------|------|
| 输入校验 | 服务端白名单校验 | 所有用户输入进行类型、长度、格式校验 | `07` §4、`09` §8 |
| SQL 注入防护 | 参数化查询 / ORM | 当前 JSON 存储不涉及 SQL；存储演进后需防范 | `08` §5.1 |
| XSS 防护 | 输出转义 + CSP | 研报内容与 LLM 回答渲染时防止脚本注入 | `07` §4 |
| CSRF 防护 | 教学版：前后端分离 REST API 使用 JSON Content-Type 天然免疫；生产版：SameSite Cookie + CSRF Token | 状态变更接口需 CSRF Token（见下表） | `07` §4 |
| 速率限制 | 教学版无限制；生产版 100 req/min/user | 防止接口滥用 | `07` §4 "速率限制：无" |
| 文件上传校验 | 类型白名单 + 大小限制 | 仅允许 PDF/HTML，限制单文件大小 | 安全最佳实践 |

### 4.1 端点级安全分析（对齐 `09` §1 全部 6 个端点）

| # | 端点 | 方法 | 状态变更 | CSRF 风险 | 输入校验要点 | 来源 |
|---|------|------|----------|-----------|-------------|------|
| 1 | `/capabilities` | GET | 否 | 无 | 无请求体，无需校验 | `09` §1 #1 |
| 2 | `/ask` | POST | **是** | 教学版免疫（JSON）；生产版需 Token | `query` 1-500 字符 + `session_id` 非空 | `09` §3、§8 |
| 3 | `/sessions` | GET | 否 | 无 | 无请求体，无需校验 | `09` §1 #3 |
| 4 | `/sessions` | POST | **是** | 教学版免疫（JSON）；生产版需 Token | `title` 可选，≤ 23 字符 | `09` §4 |
| 5 | `/sessions/<id>` | DELETE | **是** | 教学版免疫；生产版需 Token | `session_id` 路径参数必须存在 | `09` §6 |
| 6 | `/sessions/<id>/records` | GET | 否 | 无 | `session_id` 路径参数必须存在 | `09` §7 |

> **CSRF 防护总结**：3 个状态变更端点（POST /ask、POST /sessions、DELETE /sessions/<id>）均使用 JSON `Content-Type` 提交，浏览器不会通过表单自动发起 JSON 请求，教学版天然免疫 CSRF。生产版需额外增加 `SameSite=Strict` Cookie 属性和 CSRF Token 双重防护。来源：`07` §4 CORS 全开放。

## 5. LLM / AI 安全

> 针对大模型集成的专项安全措施。投研场景数据敏感，需严格管控数据流向。

| 风险点 | 防护措施 | 验收标准 |
|--------|----------|----------|
| 数据外泄至第三方模型 | 本地部署模型 / 内网百炼服务，禁止调用外网 API | 网络出口审计零外传记录 |
| Prompt 注入攻击 | 系统 Prompt 隔离，用户输入预处理过滤 | 构造恶意 Prompt 无法绕过系统指令，测试用例覆盖常见注入模式 |
| 模型幻觉 / 错误信息 | 回答标注来源引用，标识置信度 | 所有回答携带 `answer_source` 字段，用户可追溯原文；信息提取匹配度 > 90%（对齐 03 §4） |
| 敏感信息泄露（模型回复） | 输出过滤层，正则匹配敏感信息并脱敏 | 模型回复中不包含 API Key、内部 IP、个人手机号等敏感信息 |

## 6. 审计与日志

> 定义安全审计日志的记录范围、保留策略。

| 事件类型 | 记录内容 | 保留期限 | 告警 |
|----------|----------|----------|------|
| 登录 / 登出 | 用户ID、时间、IP、结果 | 180 天 | 连续失败 ≥5 次告警 |
| 研报访问 | 用户ID、文档ID、操作类型 | 365 天 | 异常批量下载告警 |
| 数据导出 | 导出内容摘要、目标 | 365 天 | 每次导出需审批 |
| 系统配置变更 | 变更人、变更前后值 | 永久 | 关键配置变更即时通知 |

## 7. 安全威胁清单（STRIDE 简表）

> 基于 STRIDE 模型，列出主要安全威胁与应对方案。

| 类型 | 威胁 | 风险等级 | 应对措施 | 负责人 | 来源 |
|------|------|----------|----------|--------|------|
| S | 伪造用户身份访问研报 | **高** | 强认证 + Token 校验 | 后端开发 | `07` §4 认证 |
| T | 研报内容在传输中被篡改 | 中 | TLS + 文件哈希校验 | 运维 | §3.2 加密策略 |
| I | 内部数据通过AI模型泄露 | **高** | 本地部署 + 输出过滤 | AI 工程师 | §5 LLM 安全 |
| D | 大量请求导致服务不可用 | 中 | 速率限制 + 三级降级策略 | 后端开发 | `07` §4 速率限制 |
| E | 普通用户提权访问管理功能（如 DELETE /sessions/<id>、POST /ask） | 中 | §2.2 角色权限矩阵 + 端点级鉴权中间件，状态变更接口强制校验角色 | 后端开发 | `07` §4、`09` §1 #2/#5 |
| R | 用户否认执行过敏感操作（如 DELETE /sessions/<id> 级联删除、POST /ask 问答记录） | 低 | traceId 全链路审计日志（§6），所有状态变更接口记录操作人 + 时间戳 + traceId，操作不可抵赖 | 后端开发 | `07` §3.1 traceId、`09` §2 |

## 8. 部署安全

> 定义部署架构中的安全要求。

- [ ] 生产/开发环境**严格隔离**，数据不互通
- [ ] 数据库访问仅限应用服务器 IP 白名单
- [ ] 敏感配置（API Key、密码）通过环境变量注入，不入代码仓库
- [ ] 依赖包定期安全扫描（`pip audit` / `npm audit`）
- [ ] `.env` 文件加入 `.gitignore`，禁止提交到代码仓库
