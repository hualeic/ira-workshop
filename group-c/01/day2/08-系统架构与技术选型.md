## 1. 架构总览

### 1.1 系统上下文

```
┌─────────────────────────────────┐
│       用户浏览器 React SPA       │
│       http://localhost:5173     │
└──────────────┬──────────────────┘
               │ HTTP  /api/v1/agent/*
               ▼
┌─────────────────────────────────┐
│     Flask Backend (WSGI)        │
│     http://localhost:5000       │
│  ┌────────────────────────────┐ │
│  │  agent_bp — 6 个 API 端点  │ │
│  │    ↓          ↓         ↓  │ │
│  │ CoPawAgent  Storage  _ok() │ │
│  └────┬──────────┬────────────┘ │
│       ▼          ▼              │
│  ┌────────┐ ┌──────────┐       │
│  │ LLM 层 │ │ 文件系统  │       │
│  │CoPaw   │ │ ./data/  │       │

└─────────────────────────────────┘
```

### 1.2 架构风格

> 选择架构风格时思考：教学场景需要什么样的复杂度？分层带来什么好处？

| 决策 | 选型 | 理由 |
|------|------|------|
| **前后端分离** | React SPA + Flask REST API | 教学清晰度：前端独立开发服务器 |
| **单体部署** | 前后端各一个进程 | 不引入微服务复杂度 |
| **分层架构** | Route → Agent → Storage | 替换存储引擎时上层零改动 |
| **三级降级编排** | CoPaw → 百炼 → Demo 链式降级 | 保障系统始终可用，外部 LLM 不可用时自动兜底（对齐 03 §5 风险应对） |

## 2. 后端分层结构

> 每层写清"允许做什么、禁止做什么"。

| 层 | 文件 | 允许做 | 禁止做 |
|----|------|--------|--------|
| **Route** | `agent_bp.py` | 参数校验、HTTP 状态码、traceId | 直接读写文件、调用 LLM |
| **Agent** | `agent.py` | 降级编排、结果组装 | 感知 HTTP 请求/响应 |
| **Provider** | `copaw_bridge.py` / `bailian_qa.py` | HTTP 调用外部 API | 操作本地存储 |
| **Storage** | `storage.py` | 文件 CRUD、数据校验 | 调用外部 API |

## 3. 前端架构

| 项 | 选型 | 理由 |
|----|------|------|
| UI 框架 | React（Hooks）≥ 18 | 教学通用性最高 |
| 构建工具 | Vite ≥ 5 | 极快 HMR，零配置启动 |
| 状态管理 | `useState` + `useEffect` | 8 个 state 变量足够 |
| HTTP 客户端 | `fetch` API | 浏览器原生，不引入 axios |
| 样式方案 | CSS Modules / 内联样式 | 教学项目无需引入 Tailwind 等重型框架，CSS Modules 避免全局冲突且零依赖 |

## 4. 三级降级编排

> 对齐 `07` 1.2 降级策略。画出降级链路，标明每级的 `answer_source` 和 `llm_used`。

```
ask(query, session_id)
  ├─[1] CoPaw  → {source:"copaw",  llm:true}
  ├─[2] 百炼   → {source:"bailian",llm:true}
  └─[3] Demo   → {source:"demo",   llm:false}
```

| 提供商 | 文件 | 配置检测 | 超时 | 错误处理 |
|--------|------|----------|------|----------|
| **CoPaw** | `copaw_bridge.py` | `IRA_COPAW_*_URL` 非空 | 20s | 返回 None 静默降级 |
| **百炼** | `bailian_qa.py` | `DASHSCOPE_API_KEY` 非空 | 120s | 区分多类错误码 |
| **Demo** | `agent.py` | 始终可用 | 0s | 纯字符串拼接 |

## 5. 数据存储架构

### 5.1 存储演进路线

| 阶段 | 存储 | 变更范围 | 说明 |
|------|------|----------|------|
| **P0（当前）** | JSON 文件 | — | 适合教学 |
| P1 | SQLite | 仅替换 Storage 层实现 | 支持并发读写与事务，适合小规模生产 |
| P2 | PostgreSQL | Storage 层 + 部署配置 | 生产级关系型数据库，支持高并发与数据分析 |

## 7. ADR（架构决策记录）

> 每条 ADR 记录一个"为什么选 A 而不选 B"的决策。至少写 2 条。

**示例 — ADR-001**：为什么选 Flask 而非 FastAPI？Flask 学习曲线低，无需 async/await，学员 Python 基础即可上手。

| ADR | 决策 | 核心理由 |
|-----|------|----------|
| ADR-001 | Flask 而非 FastAPI | 学习曲线低，教学更适合 |
| ADR-002 | JSON 文件存储而非 SQLite/PostgreSQL | 教学场景无需并发，JSON 可读性高，学员可直接查看数据文件调试 |
| ADR-003 | 前端单文件 SPA 而非多组件拆分 | 教学项目功能单一，8 个 state 变量足够，拆组件增加理解成本 |

## 8. 安全架构

> 对齐 `07` 4 安全需求。当前教学版与生产需求的差距。

| 层 | 当前措施 | 生产改进 |
|----|----------|----------|
| 认证 | 无（开放访问） | JWT |
| CORS | 全开放 | 白名单 |
| 密钥 | 环境变量 `.env` | Vault / Secrets Manager |
| 输入校验 | Route 层参数白名单校验（query 长度、session_id 非空） | WAF + 参数化 ORM |
